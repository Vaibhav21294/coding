This is how I would mentally organize topics if I were preparing for software engineering interviews today. It separates:

What stores data → Data structures
How to traverse → DFS/BFS/Recursion
How to solve efficiently → Patterns
How to optimize → DP/Greedy

Once you think this way, interview problems become easier to classify. For example:

3Sum
= Array + Sorting + Two Pointers

Number of Islands
= Grid + DFS

Top K Frequent
= Hash Map + Heap/Bucket Sort

Merge Intervals
= Sorting + Intervals

Subsets
= Recursion + Backtracking

House Robber
= Dynamic Programming
